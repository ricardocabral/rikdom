# Country And Tax Metadata Layers

## Parent PRD

Initial rikdom PRD (Codex thread, 2026-04-20)

## What to build

Introduce optional country/tax metadata conventions (without hardcoding tax engines) so users can annotate holdings for jurisdiction-specific analysis.

## Acceptance criteria

- [ ] Tax metadata conventions documented for holdings and asset types.
- [ ] Example profiles for at least 2 countries.
- [ ] No breaking change to existing portfolio files.

## Blocked by

- Blocked by #1 if schema namespace changes are required.

## User stories addressed

- Country-specific asset modeling.
- Long-term extensibility without schema bloat.
