# Security Policy

## Supported Versions

Security fixes are provided for the latest release on the `main` branch.

| Version        | Supported |
| -------------- | --------- |
| latest         | yes       |
| older releases | no        |

## Reporting A Vulnerability

Please do not report security issues in public GitHub issues.

Preferred channel:

- Use GitHub's private vulnerability reporting (Security Advisories) for this repository: https://github.com/ricardocabral/rikdom/security/advisories/new

Fallback channel:

- Open a minimal public issue requesting a private contact path if private vulnerability reporting is unavailable.

Please include:

- Affected version/commit
- Reproduction steps or proof of concept
- Impact and suggested remediation (if known)

## Vulnerability Disclosure Process

We will acknowledge receipt within 7 days and provide a status update within 14 days when possible.

We aim to fix confirmed vulnerabilities promptly and coordinate disclosure with the reporter. Unless a different disclosure timeline is agreed, we target public disclosure within 90 days after confirmation or after a fix is available.
